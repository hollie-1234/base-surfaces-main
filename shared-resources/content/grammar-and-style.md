# Wise Design: Grammar and Style Guide

## Overview

"We've got our own way of writing certain things, and we cover them here. It helps us to be consistent, whatever and wherever we're writing."

---

## A

### Acronyms
- Avoid acronyms unless commonly used and necessary
- Use full terms if spelling them out prevents confusion
- Example: PDF (Portable Document Format)

### Active Voice
- Prioritize active voice over passive voice
- Makes it clear who performs the action
- ✓ "Jon broke the sauna"
- ✗ "The sauna got broken by Jon"

### Ampersands
- Use "and" instead of "&" in standard text
- Exception: brand names like Tiffany & Co.
- ✓ "Send and receive money"
- ✗ "Send & receive money"

---

## B

### Brackets
- Use only for abbreviations or data references
- Partial sentences: punctuation outside brackets
- Complete sentences: punctuation inside brackets
- ✗ Don't place punctuation inside brackets for partial sentences

### Bullet Points
- One point per bullet
- No full stops at end of bullets
- Capitalization rule: Match the introductory sentence
  - Full sentence intro → capitalize bullets
  - Fragment intro → lowercase bullets

---

## C

### Cases
- Use sentence case for body copy, headings, subheadings, links, buttons
- Only first letter of first word capitalized
- Proper nouns (names, places) always capitalized
- ✓ "Sending money internationally"
- ✗ "Sending Money Internationally"

### Change vs. Edit
- **Change**: switching between different options
- **Edit**: making modifications to existing content
- ✓ "Change your default currency" (switching options)
- ✓ "Edit your address" (modifying existing data)

### Choose vs. Select
- **Choose**: decision-making where multiple options are valid
- **Select**: picking one option from a list (only one correct)
- ✓ "Choose how people can pay you"
- ✓ "Select your country of residence"

### Contractions
- Use contractions for conversational tone
- ✓ "We're, you're, can't, doesn't, won't"
- ✗ "We are, you are, cannot, does not, will not"

### Countries
- Abbreviate commonly abbreviated countries
- No periods between letters
- ✓ UK, US, UAE
- ✗ U.K, U.S, U.A.E

### Currencies
- Write currency codes in brackets after full name
- Always capitalize country, never capitalize currency name
- Place code after amount with space
- Avoid symbols unless market-specific
- ✓ "10,000 SGD, 50 BRL"
- ✗ "10,000sgd, 50brl"

---

## D

### Dashes
- **Em dash (—)**: emphasis, interruptions, replacing commas (spaces either side)
- **En dash (–)**: number ranges
- ✓ "Use an em dash to show emphasis — like this"
- ✓ "Range from 1–10"
- ✗ "Range 4—8" (wrong dash type)

### Dates
- Follow regional format (UK: dd/mm/yyyy)
- Don't use ordinal suffixes (-th, -rd, -nd)
- Include day-of-week for longer-form content if relevant
- ✓ "1 March" / "Wednesday 1 March 2023"
- ✗ "1st March" / "1st March 2023"

### Driving Licence
- British English spelling: "licence"
- ✓ "Driving licence"
- ✗ "Driver's licence" or "drivers licence"

### Dropdown
- One word, no hyphen
- ✓ "Dropdown"
- ✗ "Drop-down"

---

## E

### Edit
- Use when enabling modifications to existing items
- ✓ "Edit personal information"
- ✗ "Change personal information" (if editing, not switching)

### E.g.
- Replace with "for example" or conversational "like"
- ✓ "For example, like"
- ✗ "E.g."

### Emoji
- Avoid entirely—hard to localize and inaccessible
- "Our language is bold and simple. Written to be understood by anyone, anywhere."

### Ellipses
- No spaces before or after (...)
- Indicates unfinished thoughts or missing content
- ✓ "I'm not sure about it...but I'll work it out"
- ✗ "This doesn't look right ... because it isn't"

### Email
- No hyphen
- ✓ "Email"
- ✗ "E-mail"

### Etc.
- Avoid in external communications
- Provide specific examples instead
- Guidance: "Don't be lazy, give examples"

### Exclamation Marks
- Use sparingly (inconsistent with tone of voice)
- Max one at a time
- ✓ "Hey!"
- ✗ "Hey there!!!"

---

## F

### Full Stops (Periods)
- Use generously to break up text
- Don't use in headings/subheadings (product or marketing)
- Use only in body copy or as specified
- Test: if you need breath before sentence end, add more full stops

---

## L

### Login
- **Login** (one word): adjective
- **Log in** (two words): verb
- ✓ "Put in your login details" / "Log in to your account"
- ✗ "Put in your log in details" / "Login to your account"

---

## M

### Markup
- **Markup** (one word): noun
- **Mark up** (two words): verb
- ✓ "No markups on the rate" / "We don't mark up exchange rates"
- ✗ "No mark ups" / "We don't markup"

---

## N

### Numbers
- Use numerals, not spelled-out words
- Exception: if starting a sentence, spell it out
- Large numbers: use k, m, bn (lowercase, no space after)
- Currency: write full number with code
- Ranges: use "From/To" or "Between/And" (not dashes)
- ✓ "There are 9 offices" / "3 to 5 working days"
- ✗ "3–5 working days" / "1–7 October"

---

## O

### Oxford Comma
- Use only when clarity requires it
- More common in some countries—check with localization team
- ✓ "Two employees, an accountant, and a director" (if needed)
- ✓ "Two employees, an accountant and a director" (if clear)

---

## P

### Percentages
- Use % symbol, not word
- ✓ "You're about 75% of the way"
- ✗ "25 percent left"

### Perspective
- Use second person in product screens (you, your, yours, yourself)
- Use first person when writing as Wise (we, our)
- Exception: Help article titles/headings can use first person
- ✓ "Your address"
- ✗ "My address" (in product)

---

## S

### Select
- Use when user picks one option from a list (only one valid)
- Use instead of "click" for inclusivity
- ✓ "Select 'Add new'" / "Select your country of residence"
- ✗ "Click 'Add new'" / "Select your preferred rate"

### Slashes
- No spaces around slashes
- ✓ "BIC/SWIFT"
- ✗ "BIC / SWIFT"

---

## T

### Text
- Use "text," not SMS or text message
- ✓ "Contact us by text"
- ✗ "Send us a text message"

### Time
- Use numerals + am/pm with space
- Use 00 for on-the-hour times
- Use en dash for ranges
- Include timezone in brackets if needed
- ✓ "8:00 am, 8:45 pm" / "7:00–11:30 pm" / "7:00 am (GMT)"
- ✗ "8am, 8:45pm" / "7am-11:30pm"

---

## V

### View
- Use "view" instead of "see"
- ✓ "View recipients"
- ✗ "See recipients"

---

## W

### Wifi
- All lowercase, no spaces or hyphens
- ✓ "wifi"
- ✗ "Wi-Fi" or "WIFI"

### Wise
- **Third person** when discussing the brand: "Wise offers..."
- **First person** when speaking as the brand: "We need to see..."
- ✓ "Send money abroad with Wise" / "We need to see your photo ID"
- ✗ "You will show your photo ID to Wise" (awkward perspective mixing)
