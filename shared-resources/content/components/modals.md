# Modal Component — Writing Guidelines

## Title Guidelines

### ✓ Do
- Keep it to just a few words
- Put the most important words first
- Summarize or introduce the content
- Use statements rather than questions (unless confirming an action)
- Apply sentence case with no period

### ✗ Don't
- Create lengthy titles that exceed a few words
- Use questions as titles unless seeking user confirmation
- Violate sentence case rules

## Body Content Guidelines

### ✓ Do
- Keep supporting information brief — aim for a few sentences maximum
- Use complete sentences with proper punctuation
- Align terminology with the title and call-to-action
- Prioritize information by importance
- Include vital details for warning modals in the title and button, not just body text

### ✗ Don't
- Assume users will read the full content — warnings risk being skipped
- Hide critical information in body text that users might skip

## Button Label Guidelines

### ✓ Do
- Clearly describe the action's purpose in 1-2 words
- Start with strong verbs like "Pay," "Send," or "Log in"
- Echo language from the modal's title when using a primary button
- Apply sentence case with no period
- Avoid first-person pronouns ("me," "my," "I")

### ✗ Don't
- Use generic labels that create ambiguity
- Repeat secondary button language in primary actions
- Confuse users about action consequences

## Example Structure

**Title:** "Remove currency"

**Body:** "This will delete your GBP balance and all transaction history. You can't undo this action."

**Buttons:** "Remove" (primary) / "Cancel" (secondary)
